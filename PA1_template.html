Loading and preprocessing the data

This data is forked from the github repository for the Coursera Reproducible Research course. This repository includes the data itself. This processing file assumes you have downloaded the activity.zip data file, and it resides in your working directory.

To load and pre-process the data the following steps are taken:

  • activity.zip is unzipped if necessary (if the data file is not in the working directory).
  • The data from activity.csv is read into a data frame called activity.

What is mean total number of steps taken per day?

To determine the mean total number of steps taken per day, the following steps are taken.

  • A subset of activity called clnActivity is created that does not contain NAs.
  • The daily sum (excluding NAs) of steps taken is calculated using tapply(), and stored in dailySteps as a array.
  • A histogram of the daily sums in dailySteps is created.
  • The overall mean of the dailySteps array is taken to find the mean total number of steps taken per day.
  • The median() function is also used to determine the median total number of steps taken per day.
clnActivity <- droplevels(activity[complete.cases(activity), ])

dailySteps <- tapply(clnActivity$steps, clnActivity$date, sum)

hist(dailySteps, col = "cyan3", main = "Histogram of Total Steps Taken Each Day", 
    ylim = c(0, 40), xlab = "Total Steps Per Day", ylab = "Days")

mn <- mean(dailySteps)

Median total number of steps taken per day

mdn <- as.numeric(median(dailySteps))

The median total number of steps taken per day is 1.0765 × 104.

What is the average daily activity pattern?

The average daily activity pattern is examined by determining the number of steps taken over each interval, averaged across all days. This is done in several steps:

  • The mean across days of the steps taken is found for each interval using tapply() and stored in an array called intrvlSteps.
  • The resulting average number of steps taken at each interval stored in intrvlSteps is plotted in a time series plot.
  • The interval that, on average across days, has the greatest number of steps is determined by using the max() function.
intrvlSteps <- tapply(clnActivity$steps, clnActivity$interval, mean)

plot(rownames(intrvlSteps), intrvlSteps, type = "l", col = "darkolivegreen3", 
    main = "Average daily activity pattern", xlab = "Interval", ylab = "Number of Steps")

Imputing missing values

Missing values for the number of steps are imputed by inserting the average steps across all days for that particular interval. The steps to execute this process are:

  • Calculate the number of missing values in the data set
  • A new data frame called intrvlMeans based on the intrvlSteps array is created.
  • intrvlMeans is merged with the activity data set in the interval variable, to create a new column called avgMeans and a new data set that includes the avgMeans called impActivity
  • impActivity is ordered by date and interval
  • Where the steps variable is 'NA', the average number of steps for that interval is copied from the avgSteps variable.
  • Extra variables are removed from impActivity and column names are added so the format matches activity
  • The daily sum (including imputed replacement values for NAs) of steps taken is calculated using tapply(), and stored in impDailySteps as an array.
  • A histogram of the daily sums in impDailySteps is created.
  • The overall mean of the impDailySteps array is taken to find the mean total number of steps taken per day.
  • The median() function is also used to determine the median total number of steps taken per day (using the impDailySteps data set).

Are there differences in activity patterns between weekdays and weekends?

Differences between weekend and weekday activity patterns are examined by subsetting the data into these two groups and creating a time series plot of the two subsets for comparison.

  • First, the date variable in the impActivity data coerced to POSIXlt.
  • Then day of the week for that observation is determined using the days() function.
  • Two separate data frames are created, one containing all weekday observations (weekdays) and one containing all weekend observations (weekend).
  • The mean steps for each interval across days is then calculated for both, and stored in the arrays wdIntrvlSteps and weIntrvlSteps.
  • Those arrays of means are used to create the data frames wd and we.
  • Those data frames are combined using the rbind() function, to create the wkSteps data set.
  • The lattice plotting system is then used to create a pair of time series plots showing the differences between weekday and weekend activity using wkSteps.
impActivity$date <- strptime(as.character(impActivity$date), format = "%Y-%m-%d")

days <- weekdays(impActivity$date)
days[days == "Saturday" | days == "Sunday"] <- "weekend"
days[days != "weekend"] <- "weekday"

impActivity$day <- as.factor(days)

weekdays <- impActivity[impActivity$day == "weekday", ]
weekends <- impActivity[impActivity$day == "weekend", ]

wdIntrvlSteps <- tapply(weekdays$steps, weekdays$interval, mean)
weIntrvlSteps <- tapply(weekends$steps, weekends$interval, mean)

wd <- data.frame(interval = as.numeric(names(wdIntrvlSteps)), avgSteps = as.numeric(wdIntrvlSteps), 
    day = "weekday")
we <- data.frame(interval = as.numeric(names(weIntrvlSteps)), avgSteps = as.numeric(weIntrvlSteps), 
    day = "weekend")

wkSteps <- rbind(wd, we)

xyplot(avgSteps ~ interval | day, data = wkSteps, type = "l", layout = c(1, 
    2), xlab = "Interval", ylab = "Number of steps")

Differences in activity patterns

Looking at the plots we can see that weekdays have a higher peak of activity, but weekends show a more consistent activity pattern over the course of the day.
